## 암호 알고리즘

: 패스워드, 주민번호, 은행계좌와 같은 `중요정보`를 `보호`하기 위해 `평문`을 `암호화된 문장`으로 만드는 절차 또는 방법 

- `해쉬(Hash)`를 사용하는 `단방향` 암호화 방식 / `개인키 및 공개키`로 분류되는 `양방향` 암호화 방식이 있다.

## 개인키 암호화 (Private Key Encryption) 기법 

: `동일한 키`로 데이터를 `암호화`하고 `복호화`한다. 

![image](https://user-images.githubusercontent.com/64796257/160951787-f8900aa9-6d09-47db-ace2-1094741d0420.png)

평문 정보 M ⇒ `암호화 알고리즘 E` & `개인키 K`를 이용해 `암호문 C`를 만듦   
⇒ 암호문 C ⇒ `복호화 알고리즘 D`와 `개인키 K`를 이용해 `평문 M`으로 바꿈

- 암호화 대상이 `n`개 일 때 필요한 `키의 개수`는 `2n`이다.
- `대칭 암호 기법` 또는 `비밀키 암호화 기법`이라 함
- 종류 
  - 블록 암호화 방식 
    - DES : 블록 64비트 / 키 56
    - SEED : 블록 128비트 / 키 128, 256
    - AES : 블록 128비트 / 키 128, 192, 256 - 미국 표준 기술 연구소 (NIST)에서 만듦
    - ARIA : 블록 128비트 / 키 128, 192, 256 - 국정원이 만듦
  - 스트림 암호화 방식 : LFSR, RC4

- 장점 : `암호화/복호화` 속도가 `빠`르고 `알고리즘`이 `단순`하고 `파일의 크기`가 `작`다
- 단점 : `사용자 증가`에 따라 관리해야 할 `키의 개수`가 `많아짐`

## 공개키 암호화 (Public Key Encryption) 기법 

: 데이터를 `암호화`할 때 `공개키(Public Key)`를 사용하고 이는 `DB 사용자`에게 공개  
  데이터를 `복호화`할 때 `비밀키(Secret Key)`는 `관리자`가 관리

![image](https://user-images.githubusercontent.com/64796257/160952765-9628e915-52a2-49c6-9743-05e3aeac2dd0.png)

평문 정보 M ⇒ `암호화 알고리즘 E` & `공개키 P`를 이용해 `암호문 C`를 만듦   
⇒ 암호문 C ⇒ `복호화 알고리즘 D`와 `비밀키 S`를 이용해 `평문 M`으로 바꿈

- 암호화 대상이 `n`개 일 때 필요한 `키의 개수`는 `n(n-1)/2`이다.
- `비대칭 암호화 기법` 이라고도 하며 대표적으로 `RSA 기법`이 있다.
  - RSA 기법 : `큰 숫자`를 `소인수분해`하기 어렵다는 사실에 기반해서 만듦. key는 메시지를 열고 잠그는 `상수`를 의미

- 장점 : 키의 분배가 용이하고 관리해야할 키의 개수가 적음
- 단점 : `암호화/복호화` 속도가 `느`리고 `알고리즘`이 `복잡`하고 `파일의 크기`가 `크`다

## 해시 (Hash) 

: 임의의 길이의 입력 데이터나 메시지를 `고정된 길이의 값 또는 키`로 변환하는 것 

- 해시 알고리즘 = 해시 함수 / 해시 함수로 변환된 값이나 키 = 해시 값 또는 해시 키 
- `무결성 검증` 뿐만 아니라 정보보호의 다양한 분야에서 활용됨
- 복호화가 거의 불가능한 `일방향 함수`에 해당됨

- 종류 
  - SHA 시리즈
  - MD5 : 블록 512비트 / 키 128비트 
  - N-NASH : 블록, 키 128비트
  - SNEFRU : 블록 512비트 / 키 128, 256 비트
