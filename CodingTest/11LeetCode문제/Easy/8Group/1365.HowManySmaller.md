[문제](https://leetcode.com/problems/how-many-numbers-are-smaller-than-the-current-number/description/)

## 정답 코드 

간단하게 생각할 수 있는 `2중 for문`을 작성함 
``` java
class Solution {
    public int[] smallerNumbersThanCurrent(int[] nums) {
        
        // nums[i]보다 큰 숫자들이 몇 개 있는지 알아내라 

        int len = nums.length; 

        int[] res = new int[len]; 

        for(int i = 0; i < len; i++) {
            int ith = nums[i]; 
            int count = 0; 

            for(int j = 0; j < len; j++) {
                if(ith > nums[j]) count++; 
            }

            res[i] = count; 

        }

        return res; 
    }
}
```

## 더 나은 코드 

- 시간복잡도가 `O(n)`만 소요된다. 
- `Counting Sort` 생각하면 된다. 

``` java
class Solution {
    public int[] smallerNumbersThanCurrent(int[] nums) {
      
        // nums[i]의 범위가 0<= ~ <=100 이기 때문에 
        // 101개의 숫자가 들어갈 수 있다고 범위를 한정할 수 있음
        int[] count = new int[101];
        int[] res = new int[nums.length];
        
        // nums[i]의 숫자 개수를 센다. 
        for (int i =0; i < nums.length; i++) {
            count[nums[i]]++;
        }
        
        // 이 과정을 거치면 
        // count[i]의 값은 i보다 작거나 같은 개수를 의미하게 된다. 
        for (int i = 1 ; i <= 100; i++) {
            count[i] += count[i-1];    
        }
        
        for (int i = 0; i < nums.length; i++) {
            if (nums[i] == 0)
                res[i] = 0;
            else 
                // 문제에서 원하는 건 nums[i]보다 작은 값만을 원했기 때문에
                // nums[i]에서 1을 뺀 값을 인덱싱했다
                res[i] = count[nums[i] - 1];
        }
        
        return res;        
    }
}
```
